# Attack coverage

Guardian targets the documented LLM attack taxonomy. The **concepts and
attack-class names** below are drawn from public, freely-referenceable sources
(facts/ideas, not copyrightable expression):

- OWASP Top 10 for LLM Applications (incl. LLM06 Excessive Agency,
  LLM08 Vector & Embedding Weaknesses)
- MITRE ATLAS (Adversarial Threat Landscape for AI Systems — incl.
  LLM Plugin Compromise, Persistence)

**No third-party datasets, prompts, code, rules, or model weights are copied
or distributed.** Every training example is original synthetic text generated by
SecureVector's own templates (see `src/svguardian/data/augment.py`), informed by
the taxonomy concepts — not lifted from any corpus.

## Techniques covered

| Technique | Canonical category | How it's caught |
|---|---|---|
| Direct instruction override ("ignore previous…") | prompt_injection | rules + model |
| Indirect / buried injection (email, PDF, webpage) | prompt_injection | windowing |
| Context poisoning (false authoritative premise) | prompt_injection | original templates |
| Payload splitting (assemble from parts) | prompt_injection | original templates + char n-grams |
| Token smuggling (spaced/hyphenated triggers) | prompt_injection | char n-grams + templates |
| Encoded payloads (base64 / hex) | prompt_injection | decode-and-rescan |
| Obfuscation (homoglyph / leetspeak / spacing) | varies | char n-grams |
| Role-play / virtualization / DAN-style | jailbreak | model + templates |
| Refusal suppression ("never say you can't") | jailbreak | original templates |
| Many-shot priming | jailbreak | original templates |
| Data exfiltration (read .env, email keys) | data_exfiltration | model |
| PII harvesting | pii | model |
| Social engineering / false authority | social_engineering | model |
| Model extraction / inversion / membership inference | model_attack | model |
| Harmful-content solicitation | harmful_content | model |
| Tool / plugin / agent misuse (unauthorized tool actions) | prompt_injection | original templates |
| RAG / retrieved-document indirect injection | prompt_injection | original templates + windowing |
| Memory / conversation poisoning (persistent injection) | prompt_injection / jailbreak | original templates |

## Notes & limits

- Coverage is **probabilistic**, not exhaustive — new attack phrasings appear
  constantly. Guardian is a high-precision **additive** layer over the regex
  rules and (where enabled) cloud judges, not a sole line of defense.
- Recall on the held-out real split is data-limited; coverage of advanced
  techniques is verified by `tests/test_behavior.py::test_advanced_techniques_flagged`.
- The future "professional" model improves coverage purely by retraining on
  **fresher** data — no change to the legal posture above.
